# Golf Piko - 极简高尔夫解谜游戏克隆版

一个纯前端实现的极简主义 2.5D 高尔夫逻辑解谜游戏，致敬并复刻了 iOS 独立游戏 **《Golf Piko》** 的经典视觉风格与关卡解谜体验。

![游戏截图](screenshot.png)

## 🎮 游戏在线游玩 / 本地运行

本项目为**零依赖单页应用**，不需要配置任何 Node.js、Vite 或其他打包环境。

### 本地游玩：
直接双击 `index.html` 即可在任意浏览器中直接运行。
或者在项目根目录使用命令行启动一个简易本地服务器：
```bash
python3 -m http.server 8000
```
然后在浏览器中打开 `http://localhost:8000` 即可。

## 🕹️ 操作说明

* **电脑端**：使用键盘的**方向键（↑ ↓ ← →）**或 **W A S D** 键控制小球。
* **移动端/触屏**：直接在屏幕上**滑屏（Swipe）**来控制小球滚动方向。
* **自动演示**：点击底部 ▶ 按钮，观看当前关卡的最佳路径自动演示。
* **规则**：
  * 滑动后小球沿着该方向一直滚动，直到撞到**沙坑/木墙障碍**、或掉入**水池**和**终点洞口**。
  * 目标：以**最少的步数**将小球送入插着红旗的洞口，同时尽可能收集紫色区域的💎钻石。

## ✨ 核心技术特色

1. **纯 CSS 变量驱动的 2.5D 视觉系统** — 通过 `box-shadow` 营造悬浮浮岛效果。
2. **边缘自适应大圆角算法 (Autotile Corner)** — 自动检测邻居为空的角，赋予平滑浮岛外形。
3. **Web Audio API 原生音效合成** — 零外部音频文件，全部动态合成。
4. **多段动作队列动画** — 支持传送门穿梭时的分段动画链。
5. **BFS 自动寻路** — 内置最短路径验证，确保每关可通关。

## 🗺️ 关卡设计思路

### 设计原则

1. **渐进式难度**：从简单的方向反弹（L1-L2）→ 门控机关（L3）→ 传送门（L4-L5）→ 组合挑战（L6-L10）。
2. **最优路径唯一性**：每关的最佳路径经过 BFS（广度优先搜索）验证，确保存在明确的最短解。
3. **钻石作为额外挑战**：部分关卡的钻石位于"陷阱路径"上（如靠近水池），提高收集难度。
4. **机制教学**：每引入一种新机制（水池、传送门、按钮/门），都会先在简单环境中单独教学。

## 🧩 游戏道具图鉴

| 外观 | 名称 | 功能说明 |
|------|------|----------|
| ⚪ | 小白球 | 玩家控制的主角，沿方向滑动直到撞墙或掉入洞/水 |
| 🟩 | 草地 | 普通地面，球可以自由滑过 |
| 🟫 | 沙坑/木墙 | 橙色斜纹方块，阻挡球前进（球停在它前面一格） |
| 🟦 | 水池 | 蓝色波纹方块，球掉入后重置回起点，不算通关 |
| 🟪 | 紫色收集区 | 紫色方块上放有 💎 钻石，球滑过时自动收集 |
| 💎 | 钻石 | 位于紫色区域上的可收集品，收集全部为"完美通关" |
| 🚩 | 洞口（红旗）| 目标终点，球滑入即通关 |
| 🔴 | 粉色按钮 | 地面上的圆形开关，球经过时自动踩下，激活对应的木门 |
| 🚪 | 受控木门 | 深色木栅栏，初始关闭阻挡球。对应按钮被踩后打开，变为可通行 |
| 🟠 | 橙色传送门 | 成对出现(t/T)，球进入后瞬间传送到配对出口，**保持原有滑动方向继续前进** |
| 🔵 | 蓝色传送门 | 成对出现(u/U)，与橙色传送门相同机制，用于需要两对传送门的关卡 |

> **按钮 + 门机制详解**：球滑动经过按钮（🔴）时，按钮被压下（变灰），对应的木门（🚪）立即打开。注意：按钮在本次滑动中不会立即让前方的门打开（门在下一次操作时才生效）。一旦按钮激活，门将永久保持打开状态。

### 地图符号说明（开发用）

| 符号 | 含义 | 球的行为 |
|------|------|----------|
| `.` | 草地 | 正常滑过 |
| `x` | 空白（无地块）| 阻挡，不可进入 |
| `s` | 沙坑/木墙 | 阻挡，球停在前一格 |
| `w` | 水池 | 球掉入后重置回起点 |
| `p` | 紫色收集区 | 正常滑过，经过时收集钻石 |
| `h` | 洞口（红旗）| 球进入后通关 |
| `b` | 粉色按钮 | 经过时激活，打开对应的门 |
| `g` | 受控木门 | 未激活时阻挡，激活后变为可通行 |
| `t/T` | 橙色传送门对 | 进入后传送到对应出口，保持惯性 |
| `u/U` | 蓝色传送门对 | 同上，第二对传送门 |
